1) UUID (универсальный уникальный идентификатор) - 128-битное число,
которое используется для уникальной идентификации объектов
в различных системах. Гарантирует крайне низкую вероятность коллизий
при использовании корректных алгоритмов генерации.

2) Архитектура неплохая, но можно лучше :)
2.1. В своей лабе я оставил понятие "транзакция" только для
взаимодействия с деньгами (снятие, пополнение, перевод).
Заморозка и разморозка - отдельные по смыслу действия, которые реализуется через
свои команды, т.к. предполагают свою особую логику (при расширении функционала
можно было бы добавить дополнительную аутентификацию, специальные полномочия
админа размораживать счета и т.д.).
2.2 Добавлено поле статуса транзакции для регистрации ошибок.
